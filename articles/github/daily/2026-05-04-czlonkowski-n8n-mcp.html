プロジェクトの概要

n8n-mcp は、強力なワークフロー自動化ツール「n8n」を Model Context Protocol (MCP) を通じて LLM エージェントに公開するためのサーバーです。これにより、Claude Desktop、Claude Code、Cursor、Windsurf などのエージェント対応ツールから、n8n 上の既存ワークフローを呼び出したり、新しい自動化を構築させたりすることが可能になります。

機能・できること

  • ワークフローの実行: AI エージェントに対して、特定の n8n ワークフローを引数付きで実行するように指示。
  • ワークフローのメタデータ取得: エージェントが利用可能なワークフローの一覧や、各ノードの入出力形式を把握。
  • 動的な自動化構築: LLM が現在の文脈に合わせて、n8n のノードを組み合わせて新しい自動化シナリオを提案・実行。
  • マルチツール連携: n8n が持つ数百のコネクタ（Notion, Slack, Google Drive 等）を、AI エージェントのスキルとして直接活用。

リポジトリの構成

Node.js で記述されており、MCP SDK を使用して n8n の API (REST) と通信するブリッジとして機能します。設定ファイルを通じて n8n の URL と API キーを紐付けるだけで動作するシンプルな設計です。

セットアップ・導入のイメージ

npx を利用して直接実行するか、MCP 対応クライアント（Claude Desktop 等）の設定ファイルにサーバーの起動コマンドを追加します。環境変数として n8n の認証情報を設定することで、エージェントがワークフローにアクセスできるようになります。

ターゲット層 / 注目ポイント

n8n を活用して業務自動化を行っている開発者や、AI エージェントに「実際の外部アプリ操作能力」を効率的に持たせたいユーザーに最適です。既存の膨大な n8n エコシステムをそのまま MCP に接続できる点が、非常に強力な注目ポイントです。

制約・リスク

n8n サーバーが API アクセスを許可している必要があります（特にセルフホスト環境）。また、エージェントに強力な権限を与えることになるため、破壊的な操作を伴うワークフローを実行させる場合は注意が必要です。

まとめ

「ノーコード自動化」の n8n と「AI エージェント」の MCP を繋ぐミッシングリンクを埋めるツールです。エージェントができることの幅を劇的に広げ、実務への適用を加速させるポテンシャルを持っています。
