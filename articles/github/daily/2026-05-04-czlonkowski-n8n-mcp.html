これは何か

n8n-mcp は、強力なワークフロー自動化ツール「n8n」を Model Context Protocol (MCP) を通じて LLM エージェントに公開するためのサーバーです。Claude Desktop や Cursor などのエージェント対応ツールから、n8n 上の既存ワークフローを直接呼び出すことが可能になります。

何ができるか

エージェントに対して特定の n8n ワークフローを引数付きで実行させたり、利用可能なワークフローの一覧を取得させたりできます。これにより、n8n が持つ数百のコネクタを AI エージェントの拡張スキルとして統合できます。

目立つポイント

ノーコード自動化の n8n エコシステムを、コードを介さずに AI エージェントにそのまま引き渡せる点が最大の特徴です。エージェントが動的にワークフローを組み合わせて新しい自動化を提案する、といった運用も視野に入ります。

セットアップや使い方の流れ

npx を利用して実行するか、MCP 対応クライアントの設定ファイルにサーバーの起動コマンドを追加します。環境変数に n8n の API キーを設定することで、エージェントがワークフローにアクセスできるようになります。

どんな人向けか

n8n を活用して業務自動化を行っている開発者や、AI エージェントに「実際のアプリ操作能力」を効率的に持たせたいユーザーに最適です。既存の自動化資産を AI 時代に最適化したい場合に強力なツールとなります。

注意点

n8n サーバーが API アクセスを許可している必要があり、セキュリティの観点から API キーの管理には注意が必要です。また、エージェントが意図しないワークフローを起動しないよう、適切な権限設定が推奨されます。

まとめ

「ノーコード自動化」の n8n と「AI エージェント」の MCP を繋ぐミッシングリンクを埋めるツールです。エージェントができることの幅を劇的に広げ、実務への適用を加速させるポテンシャルを持っています。
