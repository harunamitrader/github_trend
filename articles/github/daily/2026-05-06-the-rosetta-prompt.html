これは何か

The Rosetta Promptは、一つのプロンプトをOpenAI、Anthropic、Googleなど、各AIプロバイダーが推奨する「独自の作法」に合わせて自動的に最適化・変換するシステムです。各社の最新ドキュメントを学習したエージェントが、プロバイダーごとのベストプラクティスを反映したプロンプトを生成します。

何ができるか

入力された一つのプロンプトを、複数のLLM向けに同時に最適化します。例えば、Anthropic向けにはXMLタグを活用した構造に、OpenAI向けにはMarkdownを多用した指示へと自動変換します。また、Firecrawlを使用して最新の公式ドキュメントを自律的にスクレイピングし、知識ベースを常に最新の状態に保つ自動更新機能も備えています。

目立つポイント

最大の特徴は、ドキュメントベースの最適化エージェントです。ReAct（Reason + Act）パターンを採用したエージェントが、プロバイダー固有のドキュメントを読み込み、具体的な変更点（構造、フォーマット等）をログとともに提案します。フロントエンドにはThree.jsを使用した3Dビジュアライゼーションが採用されており、結果を視覚的に美しく表示します。

セットアップや使い方の流れ

FastAPIによるバックエンドとReactによるフロントエンドで構成されています。OpenRouterのAPIキーを設定することで動作し、ローカル環境でAPIサーバーを立ち上げてブラウザから利用します。updaterディレクトリ内のエージェントを実行することで、ドキュメントの最新化も可能です。

どんな人向けか

複数のLLMを併用している開発者や、プロバイダーごとのプロンプトエンジニアリングに苦労しているプロンプトエンジニア、そして常に最新の公式推奨手法を反映させたいエンジニアに最適です。

注意点

最適化にはLLM（デフォルトではClaude Opus等）を使用するため、実行にはAPIキーとそれに応じたコストが発生します。また、自動スクレイピング機能を利用する場合は、FirecrawlのAPIキーも必要となります。

まとめ

The Rosetta Promptは、モデルの多様化が進む現代において、「どのモデルでも最高の結果を出す」ための橋渡しとなる強力なツールです。公式ドキュメントを自動で追従する設計は、変化の激しいAI業界において非常に実用的です。
