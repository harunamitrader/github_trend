これは何か

Thunderbird（Mozilla 系メールクライアント）の開発チームが手がける、エージェントベースのコード自動改善・監視ツールです。特定の PR（プルリクエスト）やディレクトリを「ループ」機能で監視し、AI が継続的にコード品質を監視し、必要に応じて修正や提案を自律的にコミットします。

何ができるか

  • 自律型ループ (/loop): AI に対し、タスクが完了するまで（あるいは監視し続けるように）継続的な試行・改善を指示できます。
  • GitHub 連携: GraphQL 経由でプルリクエストのメタデータを読み取り、コードの意図を汲み取った高度なレビューを提供します。
  • bash/python 実行: ローカル環境や Sandbox でテストを実行し、その結果をもとに AI がコードを修正する「テスト駆動型」の自律開発が可能です。
  • マルチモデル対応: OpenAI, Anthropic, Google Gemini 等、好みや用途に合わせてバックエンドの LLM を切り替え可能です。

目立つポイント

「大規模プロダクトの開発現場」から生まれた実用性が魅力です。汎用的なチャットとしての AI ではなく、特定のエンジニアリング・ワークフローに特化しており、bash コマンドとの組み合わせによって「AI 自身がビルドを回し、エラーを直す」という実務に近い自動化を実現しています。

セットアップや使い方の流れ

  1. Node.js 環境でリポジトリをセットアップします。
  2. プロジェクトのルートに .env ファイルを作成し、各種 API キーを設定します。
  3. CLI から thunderbolt --pr [URL] のように実行するか、/loop を使って監視を開始します。
  4. AI が提案した修正内容を確認し、問題なければレポジトリへプッシュします。

どんな人向けか

GitHub での PR 管理を自動化したいチームリーダー、大規模なリファクタリングを AI に任せたいシニアエンジニア、そして「エージェントにコードを書かせ続ける」という実験的な開発スタイルに興味がある方に最適です。

注意点

AI が自律的にコードを書き換えるため、適切なテスト環境がない場所での利用は危険です。また、Mozilla 系列のツールらしく、非常に高い透明性を持ちますが、設定には GraphQL や GitHub App の権限管理に関する知識が必要になる場合があります。

まとめ

thunderbolt は、AI コーディングにおける「次の一歩」を示しています。単なるコード生成ではなく、プロダクトの文脈を理解し、プルリクエストというサイクルの中で「働き続ける」エージェントの姿を具体化しており、今後の OSS 開発の在り方を変える可能性を感じさせます。
