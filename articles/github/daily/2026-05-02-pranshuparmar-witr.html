これは何か

witr ("Why Is This Running?") は、システム上で動作しているプロセスについて「単に動いていること」だけでなく「なぜ動いているのか」という因果関係を解き明かすための診断ツールです。Rust で書かれており、Linux, macOS, Windows, FreeBSD といった主要なプラットフォームで動作します。

何ができるか

特定のポート（例: 8080）、PID、実行ファイル名、あるいは開かれているファイルパスを指定するだけで、そのプロセスがどのように起動され、何がそれを管理しているのかをツリー形式やナラティブ（物語的）なスタイルで表示します。Docker コンテナ、systemd ユニット、SSH セッション、tmux、さらには git ブランチといった実行時の文脈までを統合して解説してくれます。

目立つポイント

1. **因果関係の可視化**: プロセスの親子関係だけでなく、systemd や launchd、Docker といった管理システムを特定し、起動の背景を説明します。 2. **マルチプラットフォーム対応**: OS ごとの低レイヤーな情報（/proc, sysctl, registry 等）を抽象化し、統一されたインターフェースを提供します。 3. **強力な診断機能**: ルート権限での実行、公開インターフェースでのリスニング、バイナリの削除、メモリ異常消費などのリスクを自動で警告します。 4. **柔軟な出力形式**: TUI によるダッシュボード表示に加え、JSON、環境変数形式、ショートサマリなどの出力に対応しています。

セットアップや使い方の流れ

各プラットフォーム向けにバイナリが配布されており、Homebrew (macOS/Linux) や PowerShell (Windows) で簡単にインストール可能です。基本的な使い方は witr <ポート番号や名前> と入力するだけです。システム全体の詳細な情報を得るには sudo (管理者権限) での実行が推奨されます。

どんな人向けか

サーバー管理、デバッグ、障害対応を行うエンジニア。特に「身に覚えのないプロセスがポートを占有している」「このサービスがどこから起動されたかわからない」といった状況で、複数のツール（ps, lsof, netstat 等）を使い分ける手間を省きたい方に最適です。

注意点

診断ツールであり、プロセスの強制終了や修正を行うためのツールではありません。また、macOS の SIP (System Integrity Protection) などのセキュリティ制限により、一部のシステムプロセスの詳細にはアクセスできない場合があります。

まとめ

witr は、プロセスの背後にある「ストーリー」を読み解くためのツールです。単なる状態表示にとどまらず、実行時のコンテキストを可視化することで、エンジニアの調査時間を大幅に短縮し、システムへの理解を深めてくれます。
