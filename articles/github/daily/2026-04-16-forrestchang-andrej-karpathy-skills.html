プロジェクトの概要

著名なAI研究開発者である Andrej Karpathy 氏がSNS等で発信した「LLMがコード生成時に陥りやすい罠や欠陥」に関する知見を集約し、Claude Code 向けの設定・指示プロンプト（CLAUDE.md）へと落とし込んだ単一のファイルからなるプロジェクトです。

機能・できること

  • 意図しない破壊的変更の防止: AIが既存の正常な設定ファイルやインフラコードを無闇に書き換える癖を抑制します。
  • コンテキスト維持の強制: 長い対話でもLLMが過去の文脈を忘却せず、一貫したアーキテクチャ設計を維持するよう指示。
  • 問題解決のルーティング: バグに直面した際、推測でコードを切り刻む前に、まずはプリントデバッグなどで確実に仮説検証させてから修正を行わせる思考プロセスを定義。

リポジトリの構成

本リポジトリの核となるのは CLAUDE.md というプレーンテキストのルール指示書のみであり、極めてシンプルな構成です。

セットアップ・導入のイメージ

プロジェクトの手元にあるルートディレクトリに CLAUDE.md をコピーして配置するだけです。Claude Code （や、それに類似機能を持つ対応エッジブラウザ/エディタ）は、この指示をシステムガイドラインとして暗黙的に読み込み、自身の振る舞いを修正します。

ターゲット層 / 注目ポイント

日々のコーディングでAIエージェントを使用しており、AIの「暴走」や「幻覚（ハルシネーション）による余計なファイル修正」に悩まされているすべての開発者に対する、軽量で即効性のある解決策です。

制約・リスク

ファイル一つであっても全タスクで読み込まれるため、プロンプト容量の消費（トークンへの影響）が発生します。小規模タスクに対してはやや過剰な指示出しになる可能性もあります。

まとめ

AI開発ツールの効果を高めるには、コードそのものよりも「エージェントにどう振る舞わせるかのルール」が最も重要であることを証明する、ミニマリストでありながら強力なアプローチの好例です。
