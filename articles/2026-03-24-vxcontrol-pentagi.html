pentagiは、自律型ペンテスト基盤をかなり本気で作っている

2026-03-24 時点の GitHub Trending にいた vxcontrol/pentagi は、複雑な penetration testing を自律的に進める AI Agents system を掲げるプロジェクトだ。 README を見ると、単なるエージェント実験ではなく、かなり重い基盤志向の設計になっている。

公開日 2026-03-24
対象 repo vxcontrol/pentagi
ソース GitHub Trending / repo page

これは何か

PentAGI は、AI を使って penetration testing の工程を自律的に進めるためのシステムだ。 README の Overview では、情報セキュリティの実務者や研究者向けの柔軟な automated security testing tool と説明されている。

何ができるか

Features を見ると、かなり多機能だ。特に目立つのは次のあたり。

  • Docker ベースの隔離実行
  • 自律的に手順を決める AI agent
  • 20 以上のペンテストツール同梱
  • 長期メモリと knowledge graph
  • 専門エージェントへの委譲
  • REST / GraphQL API

目立つポイント

いちばん強いのは、「AI に試させる」だけで終わらず、隔離実行、監視、レポート、API、永続化まで入れていることだ。 セキュリティ用途なので当然だが、基盤の重さを隠していないところがむしろ本気度に見える。

セットアップや使い方の流れ

Quick Start では Docker / Docker Compose 前提で、対話型 installer による導入が推奨されている。 Linux だけでなく Windows や macOS 向けの配布もあり、Docker 権限や `docker` グループ付与に関する注意も README でかなり丁寧に書かれている。

どんな人向けか

これは一般的な AI ツールというより、ペンテスト業務やセキュリティ研究をする人向けだ。 Web 自動化や agent orchestration に興味があるだけの人には重すぎるが、攻撃面の検証を体系化したい人には刺さる。

注意点

当然ながら authorized な環境でしか使うべきではないし、Docker 権限の扱いも重い。 自律型であること自体が価値だが、同時に権限・ログ・実行範囲の管理を甘くできないタイプの repo でもある。

まとめ

pentagi は、AI エージェントをセキュリティ実務に寄せたときの基盤設計をかなり正面から作っている。 Trending 上位にいるのも、エージェントの応用先がますます具体化している流れの一部だと見ていい。
