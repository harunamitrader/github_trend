GitHub Copilot: Copilot cloud agent が署名付きコミットに対応

Copilot cloud agent が生成するコミットが GitHub 上で Verified 扱いになり、signed commits 必須の保護ルールにも適合するようになりました。

リリース日 2026-04-03
機能 Signed commits for cloud agent
ツール GitHub Copilot

主な変更点

GitHub の公式 Changelog では、2026-04-03 付で Copilot cloud agent signs its commits が公開されました。これにより Copilot cloud agent が作るコミットは GitHub 上で Verified と表示され、改ざんされていないことを確認しやすくなります。

より重要なのは、Require signed commits を有効にした branch protection rule / ruleset のあるリポジトリでも cloud agent を使えるようになった点です。従来はこのルールに適合できず、agent 自体が使えない場面がありました。

影響や使いどころ

保護ルールが厳しい組織では、agent 導入時に「署名付きコミット必須」が障壁になりがちでした。今回の更新で、監査性と agent 利用の両立がしやすくなります。特に本番系リポジトリや Enterprise 運用では実務上の意味が大きい変更です。

まとめ

この更新は派手な新機能ではありませんが、Copilot cloud agent を保護ルール付きリポジトリへ広げる前提条件の一つを埋める内容です。branch protection を厳格にしているなら確認優先度は高めです。
