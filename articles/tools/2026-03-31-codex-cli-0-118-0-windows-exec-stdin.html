Codex CLI 0.118.0: Windows sandbox と exec 入力フローを改善

Windows sandbox の proxy-only 制御、codex exec の prompt+stdin 対応、MCP 起動まわりの安定化が入った更新です。

リリース日 2026-03-31
バージョン 0.118.0
ツール Codex

主な変更点

OpenAI の公式 Codex changelog では、Codex CLI 0.118.0 が 2026-03-31 付で公開されています。今回の新機能では、Windows sandbox が環境変数頼みではなく OS レベルの egress rule で proxy-only networking を強制できるようになりました。

また、codex exec が prompt と標準入力を同時に扱えるようになり、パイプ入力しながら別の prompt をコマンドラインで渡せます。既存の CLI 自動化フローには実用的な改善です。

実務で効くポイント

app-server では device code login が追加され、ブラウザ callback に依存しない ChatGPT サインインが可能になりました。加えて、MCP startup timeout の延長や警告表示復帰、Windows での apply_patch 周辺の不安定さも修正対象です。

`.codex` の初回作成保護や bearer token の動的更新対応も含まれており、権限制御と認証の両面で一段強くなっています。

まとめ

0.118.0 は、Windows 利用者と CLI 自動化ユーザーの双方に効くリリースです。特に proxy 制約下の運用や stdin パイプを多用するワークフローでは差分が出やすい更新です。
