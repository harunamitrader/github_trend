GitHub Copilot: Organization runner controls for cloud agent

Copilot cloud agent の実行先 runner を、organization 単位で統制できるようになった更新です。

リリース日 2026-04-03
機能 Organization runner controls
ツール GitHub Copilot

主な変更点

GitHub Changelog の Copilot ラベルでは、2026-04-03 の最新項目として Organization runner controls for Copilot cloud agent が掲載されています。リポジトリ単位ではなく organization 側で cloud agent の runner 利用方針を決めやすくする更新です。

影響や使いどころ

大きい組織では、どの runner を agent 実行に使わせるかを各リポジトリ任せにすると、ネットワーク、コスト、セキュリティの統制が崩れやすくなります。organization runner controls が入ることで、管理者側で標準構成を揃えやすくなります。

同日の signed commits 対応と組み合わせると、cloud agent を本番寄りのリポジトリへ展開する際の管理面が一段整いました。

まとめ

この更新は UI の派手さよりも、組織運用で効くガバナンス強化です。Copilot cloud agent を複数チームへ展開しているなら確認優先度は高めです。
