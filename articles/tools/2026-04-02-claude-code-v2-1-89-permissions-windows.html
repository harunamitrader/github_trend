Claude Code v2.1.89: 権限制御と Windows 安定性をさらに改善

defer 付き権限制御、MCP 接続待ちの短縮、Windows / PowerShell の不具合修正がまとまって入った更新です。

確認日 2026-04-02
バージョン v2.1.89
ツール Claude Code

主な変更点

Anthropic の公式 changelog 先頭に v2.1.89 が追加されました。今回の更新では、PreToolUse フックに "defer" を返せるようになり、ヘッドレス実行をツール呼び出し地点で一時停止し、-p --resume で再評価できるようになっています。

あわせて、PermissionDenied フックや MCP_CONNECTION_NONBLOCKING=true が入り、Auto mode の拒否後リトライや、MCP サーバー待ち時間の抑制が強化されました。権限制御を組み込んだ運用フローを組みやすくした更新と見てよさそうです。

実務で効くポイント

Windows まわりでは、Edit / Write ツールの CRLF 二重化、PowerShell 5.1 で git push の stderr 進捗を失敗扱いしていた問題、Shift+Enter の改行不具合などが修正されています。既存の Windows 利用者には体感差が出やすい内容です。

そのほか、巨大 JSON に起因するメモリリーク、StructuredOutput の schema cache 不具合、長時間セッションでの prompt cache miss、巨大セッションファイル編集時のクラッシュなども解消対象に含まれます。日常の継続運用に寄ったリリースです。

まとめ

v2.1.89 は、権限フックの柔軟性と Windows 安定性を同時に底上げする更新です。特に PowerShell 環境や headless 実行を組んでいるチームでは優先度が高めです。
