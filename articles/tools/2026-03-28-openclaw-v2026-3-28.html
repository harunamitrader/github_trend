OpenClaw v2026.3.28: xAI強化とプロバイダーの刷新

xAIの検索機能ネイティブ対応や、Qwen・MiniMaxなどの提供プロバイダー周りの大幅な整理が行われました。

リリース日 2026-03-28
バージョン v2026.3.28
ツール OpenClaw

主な新機能・変更点

xAI (Grok) 連携の強化

バンドルされている xAI プロバイダーが Responses API へと完全移行し、Grokの強力な x_search 機能がネイティブで第一級ツールとしてサポートされました。今後はWeb検索設定から自動的に有効化されるようになり、設定のセットアップフローも簡略化されています。

MiniMax による画像生成サポート

MiniMax の image-01 モデルを活用した画像生成に対応しました。このアップデートにより、テキストからの新規画像生成、画像編集、およびきめ細かなアスペクト比制御が OpenClaw から直接行えるようになります。

プラグイン/フックの拡張 (requireApproval)

開発者向け機能として、before_tool_call フックにおいて非同期の requireApproval が使用可能になりました。これにより特定のツール（破壊的操作など）が発火する直前に、UI上のオーバーレイや連携している外部クライアント（Discord, Telegram等）へユーザー承認を強制で求めるプラグインが作成可能になりました。

注意すべき破壊的変更 (Breaking Changes)

  • Qwen プロバイダーの移行: 従来の qwen-portal-auth (portal.qwen.ai の OAuth 連携) は非推奨および廃止となりました。今後は openclaw onboard --auth-choice modelstudio-api-key を利用し、Model Studio の API 経由での認証が必須となります。
  • レガシー設定移行の終了: 2ヶ月以上前の古い形式の設定ファイルに対する自動移行機能が削除されました。非常に古いバージョンから直接アップデートした場合、バリデーションエラーが発生する可能性があります。

まとめ

今回のアップデートはプロバイダー周辺の大胆な刷新（特にQwenとxAI）が中心となっており、Grokネイティブ検索の採用や画像生成の拡充など、各種AIモデルの最新APIへの追従が図られています。既存ユーザーはQwenの認証切り替えに注意が必要です。
