Release Notes

Claude Code v2.1.80: クォータの可視化とプラグイン管理の柔軟性

`rate_limits` フィールドのステータスラインへの導入と、新しいプラグインソース `source: 'settings'` のサポート。

リリース日 2026-03-20
バージョン v2.1.80
ツール Claude Code

主な新機能

Claude Code v2.1.80 では、ユーザーが自身の使用状況をより正確に把握し、環境をカスタマイズするための機能が強化されました。

`rate_limits` 表示機能

ステータスラインのスクリプトにおいて、Claude.ai のレート制限使用状況（5時間及び7日間のウィンドウ）の表示を可能にする `rate_limits` フィールドが追加されました。 これにより、消費されたパーセンテージやリセット時刻を即座に確認できます。

プラグイン管理の改善

マーケットプレイス経由ではなく、`settings.json` 内にインラインでプラグインエントリを記述できる `source: 'settings'` が導入されました。 独自のプラグインをより手軽に、かつ集中的に管理できるようになります。

モデル Effort フロントマターの対応

スキルやスラッシュコマンド（/commands）の呼び出し時に、モデルの「思考の深さ（Effort）」レベルを個別に上書き指定できるフロントマターがサポートされました。 複雑な問題に対してのみ、リソースを集中させることが可能です。

改善と修正

  • リサーチプレビュー: MCP サーバーからセッションへのメッセージプッシュを可能にする `--channels` オプションを試験導入。
  • 利便性の向上: CLI ツールの使用状況をプラグインのヒントに活用できるよう改善。
  • 安定性向上: セッションの再開、ストリーミング、音声モード、リモート操作などに関連する複数のバグを修正。

まとめ

v2.1.80 は、ヘビーユーザーにとって特に恩恵の大きいアップデートです。 レート制限の可視化と柔軟なプラグイン管理により、開発フローがより透明で拡張的なものになります。
