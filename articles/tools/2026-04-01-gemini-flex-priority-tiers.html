Gemini: Flex と Priority inference tiers を preview 公開

Google が Gemini API の推論ティアを拡張し、コスト重視の Flex と低遅延重視の Priority を preview で追加しました。

リリース日 2026-04-01
機能 Flex / Priority tiers
ツール Gemini

主な変更点

Google の公式 `Gemini API Release notes` では、2026-04-01 の更新として `Flex` と `Priority` の 2 つの inference tier が preview 公開されました。既存の標準的な呼び出しに加えて、用途ごとに価格と応答性を選びやすくする更新です。

`Flex` は待ち時間に厳しくないバッチ処理や非同期処理向けで、より安価な実行を狙う位置づけです。`Priority` は低遅延とキャパシティ確保を重視したティアとして案内されており、応答速度を優先したい本番用途と相性が良いです。

影響や使いどころ

同じ Gemini でも、処理の性質に応じてティアを切り替えられるようになったのが実務上のポイントです。たとえば要約生成や埋め込み前処理のように急がないジョブは `Flex`、チャット UI や即応型エージェントは `Priority` といった整理がしやすくなります。

これにより、単にモデルを選ぶだけでなく、運用コストと体感速度の設計を API 呼び出しレベルで分ける考え方がしやすくなりました。Gemini を複数ワークロードにまたがって使うチームほど恩恵が大きい更新です。

注意点

現時点では preview 扱いなので、価格、提供リージョン、SLA、対象モデルは今後変わる可能性があります。既存構成にすぐ全面適用するより、まずはジョブ特性の違う経路で小さく比較するのが安全です。

まとめ

今回の Gemini 更新は新モデル追加ではなく、推論の運用設計を細かく分けられるようにした点が本質です。コスト最適化と低遅延の両立を考える現場では、押さえておく価値があります。
