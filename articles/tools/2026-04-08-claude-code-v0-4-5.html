Claude Code v0.4.5: Context Freezeとトークン軽量化

ファイルコンテキストをメモリに固定するfreezeコマンドと、ルーチンタスクのコスト15%削減を実現。

リリース日 2026-04-08
バージョン v0.4.5
ツール Claude Code

主な変更点

新しい `/context-freeze` コマンドが実装されました。長いタスクを自律実行させていると、Claude Codeが古いファイルの記憶を落として再帰的にエラーを起こすことがありましたが、このコマンドで重要なコアファイルを明示的にメモリ（コンテキストウィンドウ）に固定できます。また、内部API呼び出しの最適化により、プロジェクト全体に対する単純な検索や置換タスクにおけるトークン消費が約15%削減されています。

影響や使いどころ

大規模なリファクタリングを任せる際、インターフェース定義や型定義のファイルをあらかじめfreezeしておくことで、幻覚（ハルシネーション）による型エラーの混入を劇的に減らすことができます。コスト削減も合わさり、より気軽かつ安全に長時間のタスクを投げられるようになりました。

まとめ

コンテキスト管理の柔軟性が高まり、開発者がエージェントの「忘れっぽさ」を直接コントロールできる実用的なアップデートです。
