Gemini: Gemma 4 の 26B / 31B モデルを AI Studio と API に追加

Google が 2026-04-02 の Gemini API Release notes で、Gemma 4 系の新モデル `gemma-4-26b-a4b-it` と `gemma-4-31b-it` の提供開始を案内しました。

リリース日 2026-04-02
モデル Gemma 4
ツール Gemini

主な変更点

Google の公式 `Gemini API Release notes` では、2026-04-02 の更新として `gemma-4-26b-a4b-it` と `gemma-4-31b-it` が追加されました。いずれも Gemma 4 launch の一部として、AI Studio と Gemini API の両方で利用できると案内されています。

今回の更新は、前日の Flex / Priority inference tiers 追加とは別に、利用できるモデルの選択肢そのものを増やす内容です。Gemini 系のワークフローでも、より軽量寄りと高性能寄りの中間帯を試しやすくなりました。

影響や使いどころ

AI Studio と API の両方で使えるため、まず AI Studio 上で応答傾向を確認し、そのまま API 実装へ流し込む検証がしやすいのが実務上の利点です。既存の Gemini 3 系だけではコストや応答品質のバランスが合いにくいケースで、比較候補が増えます。

特に複数モデルをルーティングする構成では、Gemma 4 を下位タスクや補助推論へ混ぜる余地が出ます。要約、分類、前処理のような処理を高価なモデルから分離したい場面で検討しやすい更新です。

注意点

リリースノート上ではモデル追加の告知が中心で、用途別の性能差や制限は別ドキュメント確認が必要です。既存の本番経路へ即時置換するより、まずはレイテンシ、出力品質、コストの比較を小さく回すのが妥当です。

まとめ

2026-04-02 の Gemini 更新は、Gemini API の運用幅を広げるモデル拡充でした。前日の推論ティア追加と合わせて、モデル選択と実行コスト設計の両面で選択肢が増えています。
