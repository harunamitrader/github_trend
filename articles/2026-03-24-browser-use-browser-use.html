これは何か

browser-use は “Make websites accessible for AI agents” を掲げる Python ライブラリだ。 つまりブラウザ自動化を AI エージェントの作業面に変えるための基盤で、用途はかなり広い。

何ができるか

README ではフォーム入力、買い物、PC パーツ探索などのデモが並んでいて、 実際の Web タスクをエージェントに処理させるための土台として使われている。 custom tools、CLI、テンプレート生成、認証例まで用意されているのも強い。

目立つポイント

open source と cloud の役割分担を README 上で明確に分けているのが印象的だった。 self-host や custom tools が必要なら OSS、本番寄りの stealth や scale が欲しいなら cloud という整理で、 単なるサンプル repo よりずっと運用イメージがある。

セットアップや使い方の流れ

Human Quickstart では Python 3.11 以上を前提に `uv init && uv add browser-use && uv sync` で導入し、 必要なら API キーを `.env` に入れた上で最初の agent を実行する流れになっている。 さらに `uvx browser-use init --template default` でテンプレートも作れる。

どんな人向けか

AI エージェントに実際の Web を触らせたい人に向いている。特に、スクリーンショットや DOM ではなく、 「継続するブラウザ状態」を使って高速に反復したい人には扱いやすい。

注意点

本番で大量に回すなら、認証、CAPTCHA、メモリ消費、対象サービスの規約を避けて通れない。 README でも production では cloud API を勧めていて、ローカルだけで全部済むとは言っていない。

まとめ

browser-use は、AI エージェントにブラウザ操作を渡すためのかなり実用寄りな基盤だった。 いまの Trending で基盤系 repo が強い流れの中でも、もっともわかりやすい代表格のひとつだと思う。
